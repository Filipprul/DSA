# Grundidee
Bubble Sort ist ein einfacher, aber ineffizienter Sortieralgorithmus.
Er arbeitet, indem er immer wieder benachbarte Elemente vergleicht und tauscht, 
wenn sie in der falschen Reihenfolge stehen.
Das größte Element "wandert" dabei pro Durchlauf nach rechts, wie eine Luftblase
nach oben, daher der Name "Bubble Sort".

# Ablauf
Man hat eine Liste:
```java
[5, 1, 4, 2, 8]
```
Bubble Sort macht Folgendes:
Durchlauf 1:
1. Vergleicht 5 und 1 -> falsch herum -> tauschen
   ```java
   [1, 5, 4, 2, 8]
   ```
2. Vergleicht 5 und 4 -> falsch herum -> tauschen
   ```java
   [1, 4, 5, 2, 8]
   ```
3. Vergleicht 5 und 2 -> falsch herum -> tauschen
   ```java
   [1, 4, 2, 5, 8]
4. Vergleicht 5 und 8 -> richtig -> kein Tausch

Wichtig: Das letzte Element ist sicher das größte Element (8)

Durchlauf 2:
Der Bobble Sort wiederholt den Prozess.
...

# Wann stoppt der Algorithmus?
Wenn ein kompletter Durchlauf ohne Tausch durchgeführt wurde -> Die Liste ist sortiert.

# Zeitkomplexität
Hier muss man sehr kritisch sein:
- Best Case: O(n) -> wenn bereits sortiert und man eine Optimierung einbaut
- Avarage/Worst Case: O(n²) -> extrem langsam bei großen Datenmengen

Warum?
Weil man im schlimmsten Fall für jedes Element alle anderen nochmal vergleichen müsste.

Mathematisch:
```java
n + (n-1) + ... + 1 = n(n-1)/2  O(n²)
```

# Vorteile
- Leicht zu implementieren

# Nachteile
- Sehr Langsam bei großen Listen
- Viele unnötige Vergleiche
- Wird kaum benutzt, weil bessere Alternativen exestieren wie:
  - Merge Sort
  - Quick Sort
  - Heap Sort

# Code-Beispiel
```java
public static void bubbleSort(int[] arr){
    boolean swapped;
    do {
        swapped = false
        for (int i = 0; i < arr.length - 1; i++){
            if (arr[i] > arr[i + 1]){
                int temp = arr[i];
                arr[i] = arr[i + 1];
                arr[i + 1] = temp;
                swapped = true
            }
        }
    } while (swapped);
}
```
